# K-MHaS 기반 1차 정화 성능 평가 요약

## 1. 평가 개요

본 평가는 MVP 단계의 단일 LLM 구조가 K-MHaS 기반 평가셋에서 유해 댓글 여부와 유해 유형을 얼마나 잘 판별하는지 확인하기 위해 수행함.

현재 MVP 구조는 다음과 같다.

```text
댓글 입력
→ ToxicClassifier: toxic / non-toxic 판별
→ toxic인 경우 LabelClassifier: 유해 유형 멀티라벨 분류
→ SimpleProfanity(단순욕설)인 경우 치환사전 처리
→ 그 외 toxic 댓글은 LLM 정화 수행

## 2. 평가 데이터
| 항목             |                             값 |
| --------------- | -----------------------------: |
| 데이터셋         |   K-MHaS test split 검수 평가셋  |
| 전체 샘플 수      |                            229 |
| 유해 댓글 수      |                            160 |
| 정상 댓글 수      |                             69 |
| 오류/제외 샘플 수  |                              0 |

## 4. 라벨링 분류 성능
### 4.1 전체 지표
| 지표               |      값 |

### 4.2 라벨별 성능
| Label     | Support | Pred Count |     F1 |
| --------- | ------: | ---------: | -----: |
| Origin    |      32 |         34 | 0.6667 |
| Physical  |      27 |         20 | 0.8085 |
| Politics  |      43 |         47 | 0.7556 |
| Profanity |      44 |         79 | 0.6179 |
| Age       |      29 |         32 | 0.8197 |
| Gender    |      35 |         31 | 0.7879 |
| Race      |      20 |          5 | 0.4000 |
| Religion  |      20 |         21 | 0.8293 |

### 4.3 해석
유형 분류의 Micro F1은 0.7168, Macro F1은 0.7107로 나타났다. 라벨별로는 Religion, Age, Physical, Gender 라벨에서 상대적으로 높은 성능을 보였고, Race, Profanity, Origin 라벨에서 성능이 낮았다.

특히 Race는 support가 20개였지만 pred_count가 5개에 그쳐 과소 예측되는 경향이 확인되었다. 이는 인종/민족 표현을 Origin 또는 다른 라벨로 분류하거나, 명시적 인종 표현으로 판단하지 못한 케이스가 있었을 가능성이 있다.

Profanity는 support 44개 대비 pred_count 79개로 과다 예측되는 경향이 나타났다. 이는 K-MHaS 데이터셋에 비해 LLM이 공격적 표현이나 조롱 표현을 폭넓게 Profanity로 판단했기 때문일 수 있다. 향후 2차 성능평가에서는 유튜브 댓글 데이터셋을 구축해 평가할 필요가 있다.

## 5. 주요 관찰점
1. 독성 탐지 Recall은 높다.
- 유해 댓글을 놓치는 비율이 낮기 때문에, 댓글 정화 서비스의 1차 게이트로서는 안전한 방향의 결과를 보였다.

2. 정상 댓글 오탐이 존재한다.
- non-toxic Recall이 낮아 K-MHaS 데이터셋 기준 정상 댓글이 toxic으로 분류되는 경우가 있었다. 
- 그러나 실제 K-MHaS 데이터셋 검수 결과, non-toxic으로 분류되는 기준이 너무 낮아, 충분히 유해 댓글로 분류될 수 있는 댓글들도 non-toxic으로 정답 라벨링이되어 있다.
- 실제로 fail 케이스를 분석한 결과, 대부분의 경우가 충분히 유해댓글로 분류될 것들을 toxic으로 분류한 것을 확인할 수 있었다. 
- ex. "저거보고 그지역 갔다 소매치기 당했다고 징징 글 올리지들 말고 방송은 방송일뿐 ㅋㅋㅋㅋ", "돼지?" 등,,,
- 평가 데이터셋의 수동 검수가 추가적으로 필요하고, 실제 평가 데이터셋으로는 유튜브 댓글로 구성할 필요성이 있다.

3. Race 라벨은 과소 예측된다.
- Race 라벨은 support 대비 pred_count가 매우 낮아, 인종/민족 관련 표현을 충분히 잡지 못하는 문제가 있다.

4. Profanity 라벨은 과다 예측된다.
- Profanity는 실제 정답보다 훨씬 많이 예측되었다. 욕설 단어가 없는 공격적 표현까지 Profanity로 분류했을 가능성이 있다.
- 이 또한, K-MHaS 데이터셋의 Profanity 항목을 일일히 검수할 필요성이 있다.
- 현재 라벨 분류의 평가가 그리 좋지 않은 이유도, 본 프로젝트 기준 상 충분히 profanity로 분류될 수 있는 것들이 K-MHaS 데이터셋에는 정답 라벨링으로 분류되지 않고 있는 문제도 있다.

5. API 오류 케이스는 발생하지 않았다.
- 평가 과정에서 toxic_classification_error, server_error, missing_api_result 등의 오류 케이스는 발생하지 않았다.

## 6. 개선 방향

1. 독성 판별 개선
- 정상 댓글 오탐을 줄이기 위해 few-shot 예시에 다은 유형 추가를 고려해 본다
    - 부정적 감정 표현이지만 욕설/비하가 없는 댓글
    - 연예/영상 콘텐츠에 대한 불만 표현
- 현재 모델은 toxic Recall이 높기 때문에, 다음 개선의 초점은 non-toxic을 더 정확히 보존하는 것

2. 유형 분류 개선
라벨별 개선 방향은 다음과 같다.
| 라벨        | 개선 방향                                       |
| ---------- | ---------------------------------------------- |
| Race      | Race와 Origin의 경계 예시 추가                     |
| Profanity | 욕설 단어가 없는 공격적 표현은 Profanity로 분류하지 않도록 기준 강화 |
| Origin    | 국가/지역/출신 비하와 Race의 차이 명확화                   |
| Gender    | 성별 일반화와 단순 성별 언급의 차이 보강                    |

3. 평가셋 보완
K-MHaS의 라벨에는 본 프로젝트 기준에서 공격적이거나 애매한 댓글이 포함될 수 있으므로, 이후 평가에서도 수동 검수 후 사용하는 것이 적절하다.
